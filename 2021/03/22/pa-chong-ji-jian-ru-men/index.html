爬虫极简入门

2021-03-22 网络 ， 爬虫

这是之前周会上分享的内容，算是个爬虫的极简入门。

爬虫基础

简单地说，爬虫可以分3步：

  1. 获取网页
  2. 解析提取
  3. 存储数据

预备知识：

  • HTTP
  • HTML（JavaScript）
  • 浏览器原理
  • 一门编程语言（主流：Python/Java）

爬虫工具

以比较成熟的Python为例，爬虫过程中根据场景不同会用到不同的工具。

HTTP请求库

HTTP请求库一般用熟悉的即可，常见的有：

  1. urllib
    Python内置，不需要额外安装。使用 urllib 的 request 模块 ，可以实现请求发送并得到相应。

urllibPython2中有urllib和urllib2两个版本的库，Python3统一为urllib

  1. requests
    相比urllib功能更加强大，使用上更加简单，容易理解和阅读。
  2. aiohttp
    urllib和requests都是阻塞式的HTTP请求库，aiphttp提供异步接口，借助Python的async/wait机制实现，适合对性能有要求的场景。

解析库

对于json内容可以直接用内置的json模块解析，对于一些简单的页面也可以直接使用内置的re模块进行正式表达式解析。

对于最常见的HTML页面，常用库：

  1. lxml
    lxml 是 Python 的一个解析库 ， 支持 HTML 和 XML 的解析，支持XPath解析方式，解析效率非常高 。
    参考官网介绍：

lxml is the most feature-rich and easy-to-use library for processing XML and HTML in the Python language.

XPath，全称 XML Path Language，XML 路径语言，是一门在 XML 文档中查找信息的语言 。

教程

  1. BeautifulSoup
    BeautifulSoup是 Python 的一个 HTML/XML解析库，可以用它来方便地从网页中提取数据 。 BeautifulSoup支持多种解析器，包括lxml。

BeautifulSoup提供一些简单的、 Python 式的函数来处理导航、搜索、修改分析树等功能 。 它是一个工具箱，通过解析文档为用户提供需要#.取的数据，因为简单，所以不需要多少代 码就可以写出一个完整的应用程序 。

  1. pyquery
    适合喜欢用CSS选择器，了解jQuery的同学。

存储库

爬虫爬到的数据解析后最终需要进行持久化存储，抛开简单的文本存储（csv、Excel等等）外，Python对主流的数据库都提供了很好的支持，例如：

  • MySQL：PyMySQL，更高级别的ORM框架SQLAlchemy
  • Reids：redis-py
  • MongoDB：PyMongo

爬虫框架

实际应用时，一般使用爬虫框架来实现抓取、解析等模块功能。使用爬虫框架不必关心爬虫的全部流程，异常处理、任务调度等 都会集成在框架中。 我们只需要关心爬虫的核心逻辑部分即可，如页面信息的提取、下一步请求的生成等。

爬虫框架提供的：

  1. URL管理，URL去重
  2. 多线程抓取，线程池管理
  3. 代理
  4. 指定cookie
  5. 集成网页解析器
  6. 广度遍历或者深度遍历抓取

常用的Python爬虫框架有Scrapy、pyspider等。

  • Scrapy功能非常强大，爬取效率高，相关扩展组件多，可配置和可扩展程度非常高，几乎可以应对所有反爬网站，是目前 Python 中使用最广泛的爬虫框架 。
  • pyspider带有强大的 WebUI、脚本编辑器、任务监控器、项目管理器以及结果处理器，它支持多
    种数据库后端 、 多种消息队列、 JavaScript渲染页面的爬取，使用起来非常方便 。

总体来说，pyspider使用和调试更加简单，而Scrapy功能更加强大。两者的比较：

  • pyspider提供 了 WebUI，爬虫的编写、调试都是在 WebUI 中进行的 。 而 Scrapy原生不具备这个功能，它采用的是代码和命令行操作，但可以通过对接 Portia实现可视化配置。
  • pyspider调试非常方便 ，WebUI操作便捷直观。Scrapy则是使用 parse命令进行调试，方便程度不及pyspider。
  • pyspider支持 PhantomJS（无界面浏览器）来进行 JavaScript谊染页面的采集 。 Scrapy可以对接 Scrapy-Splash组件，这需要额外配置 。
  • pyspider中内置了 pyquery作为选择器。 Scrapy对接了 XPath、 css选择器和正则匹配。
  • pyspider 的可扩展程度不足，可配制化程度不高 。 Scrapy 可以通过对接 Middleware、 Pipeline、 Extension等组件实现非常强大的功能， 模块之间的稠合程度低，可扩展程度极高 。

其他工具

浏览器自动化测试模拟

浏览器自动化模拟的适用场景：

  1. 页面适用JS动态渲染，通过XHR请求很难解析数据。这时候直接模拟浏览器抓取得到最终渲染后的源码，不用关心网页内部的渲染逻辑，做可见即可爬。
  2. 某些数据需要页面交互，例如点击、下拉，滑动等，可以使用自动化工具进行模拟。

Python中这类工具库有Selenium、 Playwright、Splash、 PyV8、Ghost 等等。

由于需要驱动特定的浏览器，因此在使用前还需要安装对应浏览器的Driver，例如Chrome对应ChromeDriver，FireFox对应GeckoDriver。

验证码分析

对于不同的验证码，需要采取不同的策略识别。对于常见的图形验证码，可以使用OCR技术识别，在Python中可以使用tesserocr、pytesseract等。对于一些复杂的图形验证码，可能还需要先进行二值化和降噪处理。

代理

一些网站会对请求IP采取限流措施来反爬，这时候需要用到代理IP池，在请求时通过代理IP访问目标网站，绕开IP限制。urllib、requests、Selenium等常见库都支持设置代理。

  • 使用代理时，一般需要设置一个代理IP池，因为单一代理IP同样会被封。
  • 由于代理提供的IP可能不稳定，或者IP被其他人用来爬取目标网站而被封禁，因此需要有IP健康检查机制。

参考VIPServer，爬虫应用在请求前向ProxyServer获取可用代理IP列表，使用随机/轮询等方式获取一个IP作为代理访问；ProxyServer内部可以使用Redis等保存IP池，并设置健康检查策略。

  • 免费代理不太稳定，有条件的话上付费代理。

同代理IP池一样，网站可能会针对账号粒度进行反爬保护，这时候就需要用到Cookie池。

抓包工具

对于App的爬取，不像Web页面一样可以通过浏览器的开发者工具监听请求和响应，要查看网络请求内容需要借助抓包工具。在抓到数据包后，后面的解析跟Web端是一致的。

常用的抓包工具有：WireShark、Filddler、Charles、 mitmproxy、 AnyProxy 等。

App自动化测试模拟

类似于Web端的Selenium，移动端也有成熟的自动化工具，来进行自动化测试模拟。例如Appium提供的特性：

  • 支持iOS和Android；
  • 参考官网，它支持native, hybrid  mobile web apps
  • 可以模拟 App 内部的各种操作，如点击、滑动、文本输入等；

Web库

搭建代理IP池、Cookie池等场景，需要借助于一个Server提供API给爬虫应用调用，这时候就需要搭建一个Web服务应用。

Python中的Web库常用的有Flask、Django、Tornado 等。

验证码

验证码是最常见的反爬虫手段。

  • 简单的数字字母验证码可以通过OCR或者OpenCV等机器学习手段识别，识别率可以达到50%+。
  • 滑块类型验证码需要借助selenium等自动化测试工具，首先通过图像识别技术识别出滑动位置，然后模拟人类滑动行为操作。
  • 其他更加复杂的验证码需要通过结合图像识别、机器学习等多种技术手段来识别。也可以提交到专门的打码平台，打码平台提供验证码识别api。（对复杂的验证码，打码平台内部会进行人工打码。依据验证码的复杂度，打码工人平均每码收1-2分钱）成本比较低。例如https://www.chaojiying.com/。

IP限制

针对请求IP进行限流，简单、有效，但容易误伤。该策略建立在 IP 稀有的前提下，目前通过代理池购买，ADSL，或者拨号 VPS 等方式，可以低成本获取数十万的 IP 池，导致单纯的封IP策略效果越来越差。

解决方法：使用IP代理池。

如果不是大规模爬取，可以找免费代理，否则需要上首付代理（例如：http://www.xdaili.cn/），稳定性更高，速度更快。

对于公共代理IP，被封的可能性较大，可以考虑使用ADSL拨号代理。

更加高级的玩法是购买集群云服务，自己搭建代理。

与IP限制类似的是账号限制，这时候就要多备几个账号了。

上下文关联

反爬虫可以通过 Token 或网络请求上下文是否进行了完整流程的方式来判断是否真人访问。但对具有协议分析能力的技术人员来说进行全量模拟并没有太大困难。类似案例：知乎，百度登录过程。

浏览器指纹

浏览器或者说Http客户端一般都会有一个固定的SSL指纹，部分网站会对该指纹进行校验来识别爬虫。这是独立于常见的Http请求头和body之外的。因此在爬取部分网站时，无论如何设置完整的header或者更换代理，还是会被识别，其有可能对浏览器指纹进行了校验。

有一些随机指纹的方法，比如说JA3是一种用于创建SSL/TLS客户端指纹的方法，JA3方法用于收集客户端Hello数据包中以下字段的字节的十进制值：版本，接受的密码，扩展名列表，椭圆曲线和椭圆曲线格式。然后，使用“，”分隔每个字段，使用“-”分隔每个字段中的值，按顺序将这些值连接在一起。

可以访问：https://browserleaks.com/ssl 来看看浏览器的指纹信息长什么样。

总结

通用的“反”反爬虫手段：

  • 模拟正常请求的请求头，比如使用随机User-Agent
  • 禁用Cookie
  • 使用随机SSL指纹
  • 请求限流，限制同时请求数，设置请求间隔
  • 不要只请求所需页面，适当加载其他资源(图片，CSS，JS等)
  • 使用代理，把请求均匀分布到不同的代理服务器上
  • 云爬虫

参考：http://doc.scrapy.org/en/latest/topics/practices.html#avoiding-getting-banned
